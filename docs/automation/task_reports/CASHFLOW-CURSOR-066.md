## 3. Schema decision

**No migration.**

Inspection confirmed the existing `bills` table already models recurring templates with:

- `name`, `default_amount`, `due_day`, `period_bucket`, `is_variable`
- `active_from`, `active_until`, `is_active`, `recurring`
- `category`, `notes`

Split type and Teles/Nicole amounts are derived from `default_amount` using the existing `split5149` helper at template edit time. Custom split values are validated in the form but are not persisted on the template row (they apply per generated instance in C067/C068). Debt payment schedules remain separate (`bill_instances` with `bill_id: null` and `Debt:` name prefix).

### UI scope decision

| Option | Verdict |
|--------|---------|
| Inline section + dialog on Bills page | **Chosen** |
| New route for template management | Rejected — no deep-link requirement |

### Bill Templates section (Bills page)

- Lists recurring templates with status (Active / Inactive / Scheduled / Expired)
- Shows due day, period bucket, default amount, variable flag, category, notes
- **New template** opens create dialog
- **Edit** opens edit dialog
- **Deactivate** sets `is_active = false` (archive); existing instances unchanged

### Template form fields

- Name (required)
- Variable amount toggle
- Default amount (required for fixed; optional for variable)
- Teles/Nicole split with 51/49 auto-calc and custom override in form
- Due day (1–31)
- Period bucket (`1_14` / `15_eom`)
- Active from / active until
- Category, notes

### Instance source labels

Monthly bill rows show `Source: template`, `Source: manual instance`, or debt-linked messaging (unchanged).

### Boundaries preserved

- No monthly bill instance generation
- No recurring edit branching (C068)
- No variable bill warnings (C069)
- Debt-linked bill protections unchanged
- Pay & deduct cash, mark paid, dashboard formulas unchanged
